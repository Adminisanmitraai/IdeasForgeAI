## Apple-Like Design Principles

The final frontend must follow:

- calm premium interface
- generous whitespace
- soft off-white background
- subtle green-tinted radial gradients
- strong visual hierarchy
- large refined hero typography
- clear SaaS product promise
- minimal but premium navigation
- rounded cards with subtle shadows
- glass-like panels without heavy blur
- clear CTAs with strong contrast
- readable body text
- mobile-first responsive structure
- no visual clutter
- no harsh borders
- no excessive green
- no dark-heavy UI
- no generic template look

## Color Rules

Approved palette:

- page background: #f7fbf8
- surface: #ffffff
- soft green surface: #ecfdf5
- brand green: #0f8f5b
- deep text: #102318
- muted text: #607268
- soft border: rgba(15, 143, 91, 0.16)
- soft shadow: rgba(16, 35, 24, 0.08)
- premium gold accent allowed only for small highlights: #d6a84f

Color usage:
- Use green as accent, not full background.
- Keep main layout mostly white/off-white.
- Use gradients lightly.
- Avoid bright neon colors.
- Avoid heavy black sections.
- Avoid random multi-color icons.

## Typography Rules

Use system UI font stack:

- -apple-system
- BlinkMacSystemFont
- Segoe UI
- Inter fallback
- Arial fallback

Typography scale:
- Hero eyebrow: 13px, uppercase, letter spaced
- Hero title desktop: 64px to 76px
- Hero title mobile: 38px to 46px
- Section title: 34px to 44px
- Card title: 18px to 22px
- Body text: 16px to 18px
- Small label: 12px to 14px

Typography behavior:
- Use strong weight only for headlines.
- Body text should be calm and readable.
- Avoid too many uppercase labels.
- Avoid cramped line height.
- Use line-height 1.05 to 1.15 for hero titles.
- Use line-height 1.55 to 1.7 for body text.

## Layout Rules

Desktop:
- max-width: 1180px to 1240px
- centered layout
- large top hero spacing
- clear vertical rhythm
- 12-column feeling without complex grid
- cards should align cleanly

Mobile:
- single column
- header collapses cleanly
- cards stack vertically
- CTA buttons full width where useful
- hero title remains readable
- no horizontal scroll

Spacing:
- section vertical padding: 72px to 110px desktop
- section vertical padding: 44px to 64px mobile
- card padding: 24px to 36px
- border radius: 22px to 32px
- grid gap: 18px to 28px

## Component Rules

Header:
- floating white/glass surface
- rounded pill or soft bar
- logo left
- nav center or right
- CTA right
- no heavy border
- subtle shadow

Hero:
- large headline
- clear product-builder promise
- founder-friendly subtext
- primary CTA
- secondary CTA
- trust badges
- product preview visual
- no clutter

Cards:
- white or soft green-tinted surface
- subtle border
- subtle shadow
- rounded corners
- short text
- no dense paragraphs

Buttons:
- primary: green filled
- secondary: white with soft border
- hover not required in static preview
- large tap target
- rounded pill

Badges:
- small pill
- soft green background
- safety/trust labels allowed
- avoid too many badges in one row

Preview Mockup:
- should look like a premium product builder screen
- should show idea to product flow
- should not look like raw debug output
- should not mention sandbox patch in main hero
- safety notes can appear lower in trust section

## Content Rules

Final polished frontend should speak to founders:

- Turn rough ideas into polished product previews.
- Review strategy, design, and structure before generation.
- Keep approval gates before deployment.
- Build with confidence and rollback safety.

Avoid:
- internal phase names in main hero
- debug wording
- raw sandbox wording
- too many safety disclaimers above the fold
- technical backend wording for visitors

Internal safety labels may remain in a lower trust/safety section.
