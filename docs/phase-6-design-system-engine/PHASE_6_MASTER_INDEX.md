* Brand personality
* Visual style
* Typography rules
* Color rules
* Layout rules
* Component rules
* Interaction rules
* Mobile-first rules
* Accessibility rules
* Screen design guidance
* Approval checkpoint
* Next step

The user should feel:

“My product now has a clear design direction before code begins.”

---

# Phase 6 Approval Boundary

Phase 6 must require this approval before moving forward:

**Approve Design System v1.0 before moving to Pixel-Matched Conversion or Frontend Generation.**

Do not move to Phase 7 or Phase 8 without explicit approval.

---

# Next Phase

After Phase 6 is approved, IdeasForgeAI can move to either:

## Phase 7 — Pixel-Matched Converter

Use this when the user provides a reference image and wants a screen converted with high visual accuracy.

## Phase 8 — Frontend Generator

Use this when the approved design system and screen map are ready for frontend generation.
