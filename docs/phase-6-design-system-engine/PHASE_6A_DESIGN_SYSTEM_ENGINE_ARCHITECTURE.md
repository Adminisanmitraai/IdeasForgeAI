# IdeasForgeAI

## Phase 6A — Design System Engine Architecture

### Version 1.0

## Purpose

Phase 6 creates the Design System Engine for IdeasForgeAI.

The goal is to convert an approved product blueprint into a clear design direction before frontend generation begins.

Phase 6 does not generate final frontend code.

Phase 6 defines how the product should look, feel, behave, and stay visually consistent across screens.

Core mantra:

**Less UI. More Intelligence.**

---

# 3. Design System Engine Goal

The Design System Engine should answer:

* What should this product feel like?
* What visual style fits the user and product?
* What should screens look like?
* What components are needed?
* What design rules should future generators follow?
* What should be avoided?
* Is the design ready for frontend generation?

It should help IdeasForgeAI design before coding.

---

The Design System Engine receives:

* Approved Product Blueprint
* Product Strategy
* Requirements
* Screen Map
* Target Users
* Product Category

# 6. Design Positioning

Design positioning defines the overall design direction.

It should describe the product visually in one simple line.

Examples:

* Clean founder-friendly AI studio
* Premium SaaS command center
* Mobile-first farmer assistant
* Calm productivity workspace
* Trust-first finance dashboard
* Emotional relationship app
* Creative AI design studio

For IdeasForgeAI, default direction:

**Clean founder-friendly AI product studio.**

---

# 7. Brand Personality

Brand personality defines how the product should feel.

Possible traits:

* Clean
* Calm
* Intelligent
* Premium
* Trustworthy
* Minimal
* Friendly
* Fast
* Creative
* Professional
* Human
* Futuristic
* Safe

For IdeasForgeAI, recommended traits:

* Intelligent
* Clean
* Founder-friendly
* Calm
* Premium-light
* Trustworthy
* Creative but controlled

Avoid:

* Too flashy
* Too technical
* Too dark
* Too many gradients
* Dashboard clutter
* Over-designed cards
* Complex admin feel

---

# 8. Visual Style

The Design System Engine should define:

* Light mode by default
* Soft white background
* Clean cards
* Gentle shadows
* Rounded corners
* Clear hierarchy
* Minimal buttons
* Mobile-first spacing
* Calm accent color
* Product preview emphasis
* Chat-first layout

IdeasForgeAI should feel like:

A product team workspace, not a complex builder dashboard.

---

# 9. Typography Rules

Typography should support clarity.

Rules:

* Use readable sans-serif fonts
* Strong section titles
* Short labels
* Calm body text
* Avoid tiny technical text
* Avoid too many font sizes
* Avoid heavy decorative fonts

Suggested hierarchy:

* Page title
* Section title
* Card title
* Field label
* Body text
* Helper text
* Status text

Typography should make the interface feel intelligent and easy.

---

# 10. Color Rules

Color should support trust and clarity.

Recommended default:

* Light background
* White or soft green-white cards
* Deep green / teal primary accent
* Dark readable text
* Soft borders
* Muted secondary text
* Minimal warning color
* Minimal error color

Color should not dominate the product.

Use color for:

* Status
* Approval
* Primary actions
* Selected states
* Important highlights

Avoid:

* Too many bright colors
* Heavy dark mode
* Neon effects
* Red unless warning/error
* Uncontrolled gradients

---

# 11. Spacing Rules

Spacing should keep the UI clean and breathable.

Rules:

* Use generous card padding
* Keep sections visually separated
* Avoid cramped panels
* Use consistent gaps
* Keep mobile screens scrollable
* Avoid overly dense dashboard layouts

The interface should feel calm even when the output is detailed.

---

# 12. Layout Rules

Studio V3 layout principles:

* Chat-first
* Full-screen Create Mode
* Full-screen Product Preview Mode
* Preview cards should be clear
* Product Brain output should be grouped
* Bottom input should remain simple
* Avoid side-heavy complexity
* Avoid tool overload

Future generated products should follow:

* Mobile-first layout first
* Desktop responsive later
* Clear hero or entry area
* Simple primary action
* Structured content cards
* Minimal navigation
* Approval-first flows when AI generates outputs

---

# 13. Component Rules

The Design System Engine should define reusable component behavior.

Core components:

* Chat input
* Product idea card
* AI response card
* Question card
* Strategy card
* Requirements card
* Blueprint card
* Planning card
* Approval card
* Status pill
* Primary button
* Secondary button
* Input field
* Preview panel
* Memory summary
* Safe check card

Each component should have:

* Purpose
* Visual behavior
* Interaction behavior
* Mobile behavior
* Do-not-do rule

---

# 14. Interaction Rules

Interaction should feel simple and guided.

Rules:

* One main action at a time
* Approval before major generation
* Skip should be allowed
* Save Draft should be visible where useful
* Continue should move the product forward
* Revise should be easy
* No hidden destructive action
* No deployment without approval
* No build action before readiness

The user should always know:

* What happened
* What is missing
* What is ready
* What needs approval
* What happens next

---

# 15. Mobile-First Rules

IdeasForgeAI must remain mobile-first.

Rules:

* Design for narrow screen first
* Cards stack vertically
* Buttons remain thumb-friendly
* Text remains readable
* Input stays easy to use
* Preview content remains scannable
* Avoid tiny controls
* Avoid crowded multi-column layouts on mobile

Desktop can enhance the layout, but mobile must remain the base.

---

# 16. Accessibility Rules

The Design System Engine should require:

* Readable contrast
* Clear labels
* Touch-friendly buttons
* Avoid text-only status where possible
* Avoid tiny clickable areas
* Avoid confusing color-only meaning
* Clear focus states
* Simple language

Accessibility is part of trust.

---

# 17. Screen Design Guidance

For every product blueprint, Phase 6 should create a screen design guide.

It should define:

* Screen purpose
* Main user action
* Content hierarchy
* Required components
* Visual tone
* Mobile behavior
* Empty state
* Error state
* Approval state if needed

Example screens for IdeasForgeAI:

* Create Mode
* Product Preview Mode
* Design Direction Preview
* Approval Screen
* Product Memory Summary
* Future Export Center

---

# 18. Do-Not-Do Rules

Phase 6 must not:

* Redesign frozen Studio V3
* Generate final frontend code
* Start backend work
* Create database structure
* Connect Supabase
* Add authentication
* Deploy
* Add complex dashboards
* Add too many visible settings
* Turn the product into a form builder
* Ignore human approval

---

# 19. Design Readiness Logic

## Ready for Phase 6 Draft

Yes, when:

* Product blueprint exists
* Screen map exists
* Target user is known or safely assumed
* Product category is known

## Ready for Phase 6 Approval

Yes, when:

* Design direction is clear
* Brand personality is clear
* Component rules are clear
* Layout rules are clear
* Mobile rules are clear
* Do-not-do rules are clear

## Ready for Phase 7

Yes, when:

* User provides a reference image or visual target
* Pixel-matching rules are approved
* Text replacement rules are clear

## Ready for Phase 8

Yes, when:

* Design system is approved
* Screen map is approved
* Component behavior is approved
* Layout behavior is approved

For IdeasForgeAI itself, the default direction should be:

## Design Positioning

Clean founder-friendly AI product studio.

## Brand Personality

Intelligent, calm, premium-light, creative, safe, product-team-like.

Light mode, soft cards, deep green/teal accents, structured preview cards, simple chat-first creation flow.
