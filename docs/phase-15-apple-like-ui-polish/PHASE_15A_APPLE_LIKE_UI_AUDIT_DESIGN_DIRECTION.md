# Phase 15A — Apple-like UI Audit + Design Direction

Status: Completed, not frozen.

## Purpose

Phase 15A defines the Apple-like premium UI direction for IdeasForgeAI Studio V3.

This phase is audit and design-direction only.

No UI code changes are made in Phase 15A.
No backend generation is unlocked.
No deployment is added.
No provider calls are added.
No database/auth/Supabase/secrets are added.

IdeasForgeAI currently has:
- Frozen Phase 11 Builder Workspace
- Frozen Phase 12 Controlled Real Generation Safety Track
- Frozen Phase 13 Controlled Multi-File Real Generation Track
- Frozen Phase 14 Live Preview Runner Integration Track
- Studio V3 with:
  - top navigation
  - category cards
  - left workspace sidebar
  - center AI build conversation
  - right preview panel
  - local preview runner gate
  - generated output validation references

## Current UI Strengths

- Clean builder structure exists.
- Left/center/right workspace is present.
- Status badge shows IdeasForgeAI readiness.
- Safety labels are visible.
- Preview panel is functional.
- Responsive behavior exists.
- Main builder direction is correct.

## Current UI Issues

The UI is not yet fully Apple-like because:
- Header still feels heavy.
- Spacing is not fully refined.
- Sidebar needs more premium hierarchy.
- Cards need softer depth and better rhythm.
- Typography needs more consistent scale.
- Preview panel needs more product-like polish.
- Buttons need smoother hierarchy.
- Category section still feels slightly bulky.
- Mobile/narrow layout needs cleaner refinement.
- Micro-interactions are minimal.
- Overall visual system is functional but not yet luxury/premium.

## Apple-like Design Direction

IdeasForgeAI should feel:
- calm
- premium
- spacious
- intelligent
- clean
- confident
- soft
- minimal
- highly polished

Design inspiration:
- Apple product pages
- Apple Vision Pro UI softness
- Cursor workspace clarity
- Lovable-style AI builder flow
- Linear-style clean surfaces

## Visual Principles

1. Use generous whitespace but avoid dead space.
2. Use soft surfaces, not harsh boxes.
3. Use calm shadows, not heavy shadows.
4. Use rounded cards with consistent radius.
5. Use clear hierarchy: header, mode, workspace, preview.
6. Use fewer borders and better contrast.
7. Use subtle glass-like panels.
8. Keep the workspace focused.
9. Make the right preview feel like a real product preview.
10. Keep safety labels visible but elegant.

## Typography Direction

Use:
- strong headline weight
- soft subtitle tone
- compact technical labels
- consistent card headings
- readable body line-height
- reduced visual noise

Typography should feel:
- premium SaaS
- Apple-like
- clear for non-technical users
- strong enough for builders

## Color Direction

Continue using IdeasForgeAI green identity, but refine it:

Primary:
- deep green for confidence
- soft mint for surfaces
- white/near-white backgrounds
- dark charcoal text

Avoid:
- overly bright green blocks
- too many saturated badges
- harsh contrast
- noisy gradients

## Header Direction

Top header should become:
- thinner
- cleaner
- more balanced
- less crowded
- more product-like

Header controls should feel like:
- premium pills
- calm status indicators
- soft action buttons
- clear hierarchy between Preview, Share, Publish

## Hero / Create Mode Direction

The greeting section should:
- stay compact
- not dominate the first fold
- guide the user into the builder
- feel warm but professional

Category cards should:
- be slimmer
- feel clickable
- use consistent icon size
- avoid bulky white blocks
- keep horizontal rhythm clean

## Builder Workspace Direction

The builder workspace should become the main product surface.

Desktop:
- left sidebar should feel like a real project navigator
- center chat should feel like an AI build session
- right preview should feel like a live product canvas

Mobile:
- stacked layout should remain readable
- preview should not overwhelm chat
- action controls should remain reachable

- clearer
- better spaced
