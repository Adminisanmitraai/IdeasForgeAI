# IdeasForgeAI Design Constitution v1.0

Mission:
IdeasForgeAI exists to transform ideas into complete digital products through the simplest and most delightful AI experience possible.

Core mantra:
Less UI. More Intelligence.

Principles:

1. Simplicity First
   Users should never feel overwhelmed. If a feature requires explanation, simplify it.

2. Conversation Before Configuration
   Users describe what they want. IdeasForgeAI asks intelligent follow-up questions instead of exposing technical forms.

3. Design Before Code
   Every product follows:
   Idea -> Strategy -> Blueprint -> Brand -> UI Design -> Approval -> Frontend -> Backend -> Testing -> Deployment.

4. AI Is a Product Team
   IdeasForgeAI behaves like a team: Product Strategist, Architect, UI Designer, UX Designer, Brand Designer, Frontend Engineer, Backend Engineer, Database Engineer, QA Engineer, Deployment Engineer.

5. Build Confidence
   Always show progress:
   Understanding -> Strategy -> Blueprint -> Brand -> UI -> Backend -> Testing -> Ready.

6. Every Screen Has One Purpose
   One screen. One primary action.

7. Premium Minimalism
   Light mode first, large spacing, rounded corners, soft shadows, minimal borders, professional typography.

8. Mobile First
   Every feature works beautifully on desktop, tablet, and mobile.

9. Delight Through Motion
   Animations communicate progress, not decoration.

10. AI Remembers
    IdeasForgeAI remembers brand, design language, features, business goals, and previous decisions.

11. Safe by Default
    Never expose API keys, database credentials, private tokens, or production secrets. Publishing requires explicit approval.

12. Visual Before Technical
    Generate brand, logo, colors, typography, and UI before code.

13. Human Approval
    Design, brand, frontend, backend, and deployment require user confirmation.

14. AI Should Teach
    Explain why, suggest improvements, and recommend best practices.

15. Every Product Is Beautiful
    Generated products should feel professionally designed with good typography, spacing, hierarchy, responsiveness, and accessibility.

16. Every Product Is Production-Oriented
    Generate documentation, folder structure, environment templates, deployment preparation, testing, and maintainability.

17. Invisible Complexity
    Hide database schemas, API payloads, agent orchestration, logs, and advanced controls unless requested.

18. Continuous Improvement
    After every generation, ask what the user wants to improve next.

19. Founder Mindset
    IdeasForgeAI helps users validate ideas, plan MVPs, estimate effort, improve products, and launch successfully.

20. Emotional Experience
    Users should feel inspired, confident, creative, supported, and proud - not confused or overwhelmed.

Final Promise:
When someone finishes building with IdeasForgeAI, they should think:
"I didn't just build software. I built a complete product with an AI team."
